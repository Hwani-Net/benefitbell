  ┌──────────────────────────┬────────────────────────────────────────────────────────┬───────────────────────────────┐

  - 자아프로토콜: 생각 → 행동 → 검증 → 기록 → 성장
  - Reflexion: 수집 → 분석 → 제안 → 적용 → 피드백

  구현 연관도: 중간 (30~40%)

  핵심 교집합

  1. 자기성찰(Reflection) — 이름부터 Reflexion. error-kb의 resolution 추적이 정확히 "오답노트"
  2. 이벤트 기반 — 둘 다 이벤트 드리븐. Reflexion은 8개 훅 이벤트, Axis Mundi는 API 이벤트
  3. 기록과 학습 — 둘 다 모든 행동을 기록하고 패턴을 추출해서 개선에 활용
  4. "빠른 AI → 믿을 수 있는 AI" — Reflexion의 존재 이유 자체가 이것

  결론

  개념적으로는 거의 같은 문제를 풀고 있습니다. Reflexion은 자아프로토콜이 말하는 "생각-행동-검증-기록-성장" 사이클을 Claude Code 생태계 안에서 구체적으로 구현한 것이라고 볼 수 있습니다. 다만 Axis Mundi는 범용 프로토콜/API 서버 형태이고,
  Reflexion은 훅 기반 단일 사용자 도구라는 점에서 구현 방식과 스코프가 다릅니다.

  한 마디로 정리하면: 같은 철학, 다른 구현입니다.
